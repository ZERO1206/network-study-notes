# TCP 与 UDP 总结对比

---

## 核心差异对比

| | TCP | UDP |
|------|-----|------|
| 面向连接 | 是（三次握手） | 否（直接发） |
| 可靠性 | 可靠（ACK + 重传） | 不可靠 |
| 有序性 | 有序（序号重排） | 可能乱序 |
| 传输速度 | 较慢 | 快 |
| 头部大小 | 20~60 字节 | 8 字节 |
| 流量控制 | 有（滑动窗口） | 无 |
| 拥塞控制 | 有（慢启动/拥塞避免） | 无 |
| 适用场景 | 网页/文件/邮件 | 游戏/直播/语音/DNS |

---

## 各自典型应用

```
TCP 场景（数据不能丢）：
├─ HTTP/HTTPS → 网页浏览
├─ FTP        → 文件传输
├─ SMTP/POP3  → 邮件
└─ SSH/Telnet → 远程登录

UDP 场景（实时性优先）：
├─ DNS        → 快速域名查询
├─ DHCP       → 获取 IP 地址
├─ VoIP       → 语音视频通话
├─ 在线游戏   → 位置状态更新
├─ 直播/流媒体 → 实时音视频
└─ TFTP/SNMP  → 简单网络管理
```

---

## 怎么选？

```
数据一丁点都不能丢？
  是 → TCP
  否 → 延迟比丢包更致命？
          是 → UDP（游戏、通话、直播）
```

## 面试/考研常见考点

1. **TCP 和 UDP 的区别**（100% 必问）
2. TCP 如何保证可靠传输（ACK/序号/重传/流量/拥塞）
3. 为什么 DNS 用 UDP 而不是 TCP？
4. 视频通话为什么用 UDP？

---

## 关键点回顾

- TCP 头部 20~60 字节（处理多），UDP 头部 8 字节（轻量）
- DNS 用 UDP，HTTP 用 TCP
- UDP 适合：丢部分数据无所谓，延迟比丢包更致命的场景

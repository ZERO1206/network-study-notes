# 其他应用层协议

---

## 一、邮件协议三件套

### 邮件发送链路

```
发件人 → SMTP → 发件服务器 → SMTP → 收件服务器 → POP3/IMAP → 收件人
```

### SMTP —— 发邮件

- 负责发送邮件 + 服务器间转发
- 端口 25（或 587 加密）
- 不管收邮件——收邮件是 POP3/IMAP 的事

### POP3 —— 收邮件到本地

- 端口 110
- 邮件下载到本地硬盘，服务器可能自动删除
- 换台电脑就没了，越来越少见

### IMAP —— 多设备同步

- 端口 143
- 邮件始终保存在服务器，手机/电脑/平板一致
- 删一封，所有设备同步。**现在主流。**

```
SMTP = 把信寄出去
POP3 = 把信从邮局拿回家（拿走就没了）
IMAP = 在邮局看信，信永远留在邮局（多设备共享）
```

## 二、FTP —— 文件传输协议

专门上传下载文件。和 HTTP 不同——HTTP 只管"获取"，FTP 做完整文件管理（上传/下载/删/建目录）。

### FTP 的最大特点：双通道

| 通道 | 端口 | 作用 |
|------|------|------|
| 控制连接 | 21 | 发送命令（"下载 xxx"） |
| 数据连接 | 20 | 传输文件内容 |

就像对讲机两个频道——一个说指令，一个传数据，互不干扰。

### 主动模式 vs 被动模式

| | 主动模式 | 被动模式 |
|------|---------|---------|
| 数据连接谁发起 | 服务器 → 客户端 | 客户端 → 服务器 |
| 防火墙友好 | 差（客户端防火墙会拦） | 好 |
| 现在用哪个 | 基本不用 | 主流 |

## 三、常用端口速查表

| 协议 | 端口 | 传输层 | 用途 |
|------|------|--------|------|
| HTTP | 80 | TCP | 网页浏览 |
| HTTPS | 443 | TCP | 加密网页 |
| DNS | 53 | UDP(主要) | 域名解析 |
| DHCP | 67(服)/68(客) | UDP | 自动分配 IP |
| SMTP | 25 | TCP | 发邮件 |
| POP3 | 110 | TCP | 收邮件(本地) |
| IMAP | 143 | TCP | 收邮件(同步) |
| FTP 控制 | 21 | TCP | FTP 命令 |
| FTP 数据 | 20 | TCP | FTP 传输 |
| SSH | 22 | TCP | 远程登录 |
| Telnet | 23 | TCP | 远程登录(明文) |

---

## 关键点回顾

- SMTP 发邮件，POP3 收本地，IMAP 同步服务器
- FTP 双通道：21 命令 + 20 数据
- 端口是应用层协议的"门牌号"

# IPv6 基础

---

## 为什么需要 IPv6？

IPv4 只有 2³² ≈ 43 亿个地址，2011 年已分完。NAT 续了命但不是长久之计——破坏端到端，外面连不进来。根本解决方案：**IPv6**。

---

## IPv6 地址格式

**128 位**，是 IPv4 的 4 倍。2¹²⁸ 个地址——大到每粒沙子都能分配一个 IP。

改用**冒号分十六进制**，8 组，每组 4 位十六进制：

```
IPv4:  192.168.1.1                              ← 32位，点分十进制
IPv6:  2001:0db8:0000:0000:0000:ff00:0042:8329  ← 128位，冒号十六进制
```

---

## 缩写规则（必须会）

### 规则 1：每组内的前导零可省略

```
0db8 → db8
0042 → 42
0000 → 0（至少保留一个 0）
```

### 规则 2：连续全零组用 `::` 代替（只能用一次）

```
2001:0db8:0000:0000:0000:ff00:0042:8329
  ↓
2001:db8::ff00:42:8329

:: 只能出现一次——多个 :: 无法确定各省略了几组
```

---

## 常见特殊地址

```
::1           = 127.0.0.1      （本机环回）
::            = 0.0.0.0        （未指定/全零）
fe80::/10     = 链路本地地址    （类似 169.254.x.x）
2001::/16     = 全球单播地址    （类似公网 IPv4）
```

---

## IPv6 vs IPv4 关键差异

| | IPv4 | IPv6 |
|------|------|------|
| 地址长度 | 32 位 | 128 位 |
| 地址数量 | ~43 亿 | 3.4×10³⁸（海量） |
| 表示法 | 点分十进制 | 冒号十六进制 |
| NAT | 需要（地址不够） | **不需要**，每设备公网地址 |
| IPsec | 可选 | 强制支持 |
| 头部长度 | 不定长（20~60） | **固定 40 字节** |
| 校验和 | 有 | **无**（上层 TCP/UDP 有） |
| 广播 | 有 | 取消，改用组播 |
| ARP | 有 | 取消，用 NDP 替代 |

IPv6 头部**固定 40 字节** + **无校验和** → 路由器处理更快。可选字段放"扩展头部"。

---

## IPv4 → IPv6 过渡

| 方案 | 做法 | 通俗理解 |
|------|------|---------|
| 双栈 | 设备同时跑 IPv4 + IPv6 | 哪个能用用哪个 |
| 隧道 | IPv6 包封装在 IPv4 包里传 | 套个 IPv4 外壳过"IPv4 区域" |
| 翻译(NAT64) | IPv6 ↔ IPv4 互转 | 类似 NAT，转协议版本 |

---

## 关键点回顾

- IPv6 = 128 位，冒号十六进制，8 组
- `::` = 压缩连续全零组，只能用一次
- 头部固定 40 字节 + 无校验和 → 比 IPv4 快
- 过渡方案：双栈、隧道、NAT64

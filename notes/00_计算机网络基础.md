# 计算机网络基础笔记

---

## 一、什么是网络？

**网络 = 多台独立计算机连起来，让它们互相通信。**

通信需要三样东西：

- **物理连接**（网线/WiFi/光纤）
- **共同语言**（双方理解对方在说什么）
- **规则**（谁先说话、怎么确认、怎么结束）

### 网络规模的演进

```
两台直连 → 局域网(LAN) → 广域网(WAN) → 互联网(Internet)
```

---

## 二、什么是协议？

**协议 = 通信双方约定的规则。** 规定了三样东西：

| 要素 | 含义 | 电话类比 |
|------|------|----------|
| 语法 | 消息长什么样？字段怎么排？ | "喂"开头，"拜拜"结尾 |
| 语义 | 每部分什么意思？该怎么反应？ | "明天吃饭？" = 发出邀约 |
| 时序 | 谁先说话？说完了等不等回复？ | 你先"喂"，对方才能回 |

---

## 三、分层模型

### 为什么分层？

就像快递：你只管写地址，货车司机只管运箱子，快递员只管按门牌送。**每层只做自己的事**，改一层不影响其他层。

### OSI 七层 vs TCP/IP 四层

```
OSI 七层                    TCP/IP 四层        核心职责
──────────────────────────────────────────────────────
应用层 ┐
表示层 ├─→  应用层            HTTP, DNS, SMTP...
会话层 ┘                    "我要什么"
──────────────────────────────────────────────────────
传输层 ──→  传输层            TCP(可靠), UDP(快速)
                           "保证送到"
──────────────────────────────────────────────────────
网络层 ──→  网络层            IP(寻址路由)
                           "走哪条路"
──────────────────────────────────────────────────────
链路层 ┐
       ├─→  网络接口层         MAC, 物理信号
物理层 ┘                    "一步怎么走"
```

### 封装与解封装

```
发：应用层 ─包─→ 传输层 ─包─→ 网络层 ─包─→ 接口层 ─→ 网线
收：网线 ─→ 接口层 ─拆─→ 网络层 ─拆─→ 传输层 ─拆─→ 应用层
```

**为什么封装？** 每层需要自己的"说明信息"（头部）。中间路由器只拆到网络层看 IP，不关心传输层端口号和 HTTP 内容。不该知道的就原封不动包在里面。

---

## 四、各层概览

| 层 | 职责 | 核心概念 | 代表协议 |
|----|------|---------|---------|
| 应用层 | 为程序提供网络服务 | 请求/响应 | HTTP, DNS, DHCP |
| 传输层 | 数据分块、可靠送达 | 端口号 | TCP, UDP |
| 网络层 | 寻址与路由 | IP 地址 | IP, ARP |
| 接口层 | 相邻设备物理传输 | MAC 地址 | 以太网 |

**IP地址 + 端口号 = 全世界唯一标识你的某个程序**

## 五、HTTP 协议

HTTP 规定浏览器（客户端）和服务器之间怎么对话。**永远客户端先开口。**

### 请求示例

```
GET / HTTP/1.1          ← GET=获取, /=根目录, HTTP/1.1=版本
Host: www.example.com    ← 告诉服务器访问哪个网站
User-Agent: Mozilla/5.0  ← 浏览器信息
Accept: text/html        ← 想要 HTML 格式
```

### 响应示例

```
HTTP/1.1 200 OK               ← 状态码+描述
Content-Type: text/html        ← 返回内容类型

<!DOCTYPE html>...              ← 空行之后是实际内容
```

### 常见状态码

| 状态码 | 含义 | 类比 |
|--------|------|------|
| 200 OK | 成功 | "没问题，拿去" |
| 301 Moved Permanently | 永久搬家 | "新地址是xxx" |
| 404 Not Found | 不存在 | "查无此人" |
| 500 Internal Server Error | 服务器崩了 | "我这边出bug了" |

## 六、DNS

**DNS = 互联网的电话簿。把域名翻译成 IP 地址。**

### 为什么需要域名？

- IP 太难记（`110.242.68.66` vs `baidu.com`）
- IP 会变，域名不变（百度换机房只需后台更新记录）

### 访问网页的顺序

```
输入 baidu.com → DNS查IP → 拿到IP → TCP连接 → HTTP发请求
                 ↑先做这步              ↑再做这步
```

---

## 七、UDP

**UDP = 发了就不管的协议。** 没有连接、没有确认、没有重传。

### 类比

```
TCP = 挂号信（有回执，丢了重寄）
UDP = 明信片（扔邮筒不管，丢了拉倒）
```

### 为什么有 UDP？

正因为 UDP 什么都不做，它才**快**。适合实时场景：

| 场景 | 原因 |
|------|------|
| 游戏 | 旧位置数据没意义，重传反而卡顿 |
| 语音/视频 | 丢一点帧比卡顿等重传好 |
| DNS | 一次查询很短，丢了重问就行 |
| 直播 | 观众在意流畅，不在意丢几帧 |

### TCP vs UDP

| | TCP | UDP |
|------|-----|------|
| 连接 | 先建立再传 | 无连接，直接发 |
| 可靠 | 丢包重传 | 丢了不管 |
| 有序 | 不乱序 | 可能乱序 |
| 头部 | 20~60 字节 | 8 字节 |
| 场景 | 网页/文件/邮件 | 游戏/语音/直播/DNS |

## 八、TCP —— 可靠传输

TCP 解决三个问题：**丢了？乱了？太快？**

### 1. ACK + 重传（解决"丢了"）

```
发送方 ──数据──→ 接收方
       ←─ACK──     ← "收到！"
       
没收到 ACK → 超时 → 重传
```

**ACK = Acknowledgement = "确认收到"**

### 2. 序号（解决"乱了"）

TCP 给每个字节编号。接收方按序号重排，上层拿到的永远是整整齐齐的数据。

### 3. 流量控制（解决"太快"）

接收方："我还能收 XXX 字节，别发超了。"

### 三次握手（建立连接）

```
客户端 ──SYN(想连你)──────→ 服务器
       ←──SYN+ACK(好)────
       ──ACK(那我开始)───→
       
       连接建立，开始传数据
```

**为什么三次不是两次？** 防止旧 SYN 骗服务器建立废弃连接。第三次 ACK 让客户端确认"我确实要连"。

### 四次挥手（断开连接）

```
客户端 ──FIN(我说完了)───→ 服务器
       ←──ACK(知道了)────
                ...服务器可能还有数据...
       ←──FIN(我也完了)──
       ──ACK(知道了)────→
       
       连接关闭
```

**为什么四次不是三次？** TCP 是全双工——双方都能收发。客户端说完了不代表服务器也说完了，所以两个方向各关各的。

## 九、IP 协议

**IP 的核心任务：把数据从 A 送到 B，逐跳转发。**

```
你(北京) → [路由器1] → [路由器2] → ... → 百度(深圳)
             看一眼目的地  看一眼目的地
             决定下一跳    决定下一跳
```

### IP 特点

- **不可靠**：只管尽力转发，丢了不管
- 可靠是上层 **TCP** 的事（ACK+重传+序号补偿）
- 分层的好处：IP 保持简单，上层各取所需

### IPv4 地址

`192.168.1.1` = 32 位二进制，写成 4 个 0~255 的十进制数

总地址空间：2³² ≈ **43 亿**（不够用了，所以有 NAT 和 IPv6）

### TTL（生存时间）

每经过一个路由器减 1，减到 0 丢弃。**防止数据包在网络里死循环。**

## 十、ARP

**ARP = 根据 IP 查 MAC。只在局域网内有效。**

```
你广播："谁是 192.168.1.8？MAC 是多少？"
对方单播："是我！MAC = BB:BB:BB:BB:BB:BB"
其他人：不吭声（不是找我）
```

### IP vs MAC

| | IP 地址 | MAC 地址 |
|------|---------|----------|
| 层 | 网络层 | 链路层 |
| 类比 | 门牌号 | 身份证号 |
| 可变 | 可配，会变 | 出厂烧录，不变 |
| 作用范围 | 全球 | 局域网内 |
| 谁用 | 路由器 | 交换机 |

## 十一、DHCP

**DHCP = 连网时自动给你分配 IP 地址，一步到位。**

### DORA 四步

```
D: Discover  客户端广播："谁在？我要上网！"
O: Offer     DHCP 回复："192.168.1.10 给你"
R: Request   客户端确认："好，我就用这个"
A: ACK       DHCP 确认："租期 24 小时，给你"
```

### DHCP 给的四个参数

| 参数 | 作用 |
|------|------|
| IP 地址 | 你在这网的身份证 |
| 子网掩码 | 本地网络有多大 |
| 默认网关 | 出网的出口（路由器地址） |
| DNS 服务器 | 域名翻译找谁 |

缺一个都上不了网。有**租期**（通常 24h），租期过半自动续租。

---

## 十二、子网掩码

**IP = 网络号 + 主机号。子网掩码就是"切割线"。**

```
255 = 11111111（8个1）= 网络号部分
0   = 00000000（8个0）= 主机号部分

192.168.1.10  /  255.255.255.0
└─ 192.168.1 ─┘  └── .10 ──┘
   网络号              主机号
```

CIDR 简写：`192.168.1.0/24` 表示掩码有 24 个 1，即 `255.255.255.0`

**为什么分网络号和主机号？** 路由器只看网络号决定往哪转发。同网段直接传，不同网段走网关。

---

## 十三、路由器与默认网关

**默认网关 = 局域网的出口。** 数据要出网，第一步先送网关。

```
你的设备(192.168.1.5) → 网关(192.168.1.1) → 公网 → ... → 目的地
```

路由器工作：**查路由表 + 转发。** `0.0.0.0/0` = 默认路由，表里没匹配的都走这条。

---

## 完整上网流程

```
输入 www.baidu.com

① DNS     → 域名 → IP 地址
② 子网掩码 → 判断是否同网段 → 不同，走网关
③ ARP     → 找到网关的 MAC 地址
④ TCP     → 三次握手建立连接
⑤ HTTP    → 发送 GET / HTTP/1.1
⑥ 服务器返回网页 → 浏览器渲染
```

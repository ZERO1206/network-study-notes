# NAT 网络地址转换

---

## 问题背景

IPv4 总共只有 43 亿个地址，全球设备早就超过了。为什么你还能上网？**因为 NAT。**

---

## 公网 IP vs 私有 IP

互联网设计者留了三段私有地址，公网路由器不转发：

| 私有地址段 | 范围 | 常见于 |
|-----------|------|--------|
| 10.0.0.0/8 | 10.x.x.x | 公司/学校内网 |
| 172.16.0.0/12 | 172.16.x.x ~ 172.31.x.x | 较少见 |
| 192.168.0.0/16 | 192.168.x.x | **你家路由器** |

连 WiFi 拿到的 `192.168.1.10` 就是私有 IP。全世界几亿台设备 IP 跟你一样——没关系，它们在**不同的局域网**里。

---

## NAT 工作原理

**NAT = Network Address Translation（网络地址转换）**

路由器把内网私有 IP 临时换成自己的公网 IP：

```
手机(192.168.1.10) → 路由器(内网.1 / 公网1.2.3.4) → 百度
                         │
                    源地址: 192.168.1.10 → 1.2.3.4
                         │
                    百度回复给 1.2.3.4 → 路由器转回 192.168.1.10
```

**对外面来说，全家设备共享同一个公网 IP。**

### 类比：公司电话总机

```
总机号(010-12345678) = 公网 IP → 外面只认这个号
分机号101/102/103      = 内网 IP → 只有公司内部认识
前台转接               = NAT
```

## NAPT —— 多台设备同时上网怎么区分？

靠端口号。**NAPT = 内网IP+端口 ↔ 公网IP+端口**

```
内网地址:端口            ─→    公网地址:端口
192.168.1.10:54321     ─→    1.2.3.4:40001
192.168.1.11:54321     ─→    1.2.3.4:40002
192.168.1.10:56789     ─→    1.2.3.4:40003
```

即使两台设备用了相同源端口，NAT 也映射成不同公网端口。回包一到，查表分拣。

---

## NAT 优缺点

| 优点 | 缺点 |
|------|------|
| 节省公网 IP | 破坏端到端原则 |
| 对外隐藏内网结构（安全） | 外面无法主动连进来 |
| 内网随便改，不影响公网 | P2P/联机游戏需要打洞 |

**"外面无法主动连进来"** 因为私有 IP 在公网上不可路由——外部发来的包，NAT 表里没记录就丢弃。内网设备必须先往外发请求，留下记录，回包才能顺着回来。

---

## 关键点回顾

- NAT 在 **路由器** 上执行
- 转换的是：**内网IP + 端口 ↔ 公网IP + 端口**
- 私有 IP 在公网上 **不可路由**，必须通过 NAT 才能上网

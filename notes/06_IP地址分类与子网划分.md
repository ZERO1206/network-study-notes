# IP 地址分类与子网划分

---

## 一、传统 IP 地址分类（有类编址）

### A/B/C 类速查表

| 类 | 开头位 | 范围 | 默认掩码 | CIDR | 每网络主机数 |
|----|--------|------|---------|------|-------------|
| A | 0xxxxxxx | 1.0.0.0 ~ 126.255.255.255 | 255.0.0.0 | /8 | 2²⁴−2 ≈ 1677万 |
| B | 10xxxxxx | 128.0.0.0 ~ 191.255.255.255 | 255.255.0.0 | /16 | 2¹⁶−2 = 65534 |
| C | 110xxxxx | 192.0.0.0 ~ 223.255.255.255 | 255.255.255.0 | /24 | 2⁸−2 = 254 |

- **127.x.x.x** 保留（127.0.0.1 = 本机环回地址，测试用）
- **D 类** 224.0.0.0 ~ 239.255.255.255 — 组播（一对多）
- **E 类** 240.0.0.0 ~ 255.255.255.255 — 保留/实验

### 怎么看类？

**看第一个字节的二进制开头：**

```
A 类：0.......   第一位是 0
B 类：10......   前两位是 10
C 类：110.....   前三位是 110
```

**不看二进制也行，直接记十进制范围：**

```
A 类：1 ~ 126        （0 和 127 保留）
B 类：128 ~ 191
C 类：192 ~ 223
```

所以看到一个 IP，第一段落在哪里就知是哪类：
- `10.0.0.1`  → 第一段 10，在 1~126 → A 类
- `172.16.5.3` → 第一段 172，在 128~191 → B 类
- `192.168.1.1` → 第一段 192，在 192~223 → C 类

## 二、CIDR 无类编址

现实早已废弃严格 ABC 分类，改用 **CIDR（无类别域间路由）**：

```
格式：IP地址/前缀长度

/8   → 掩码 255.0.0.0       （前 8 位是网络号）
/16  → 掩码 255.255.0.0     （前 16 位是网络号）
/24  → 掩码 255.255.255.0   （前 24 位是网络号）
/26  → 掩码 255.255.255.192
```

前缀长度 = 子网掩码中连续 1 的个数。掩码不必遵守 /8、/16、/24 的边界，可以任意切。

## 三、子网划分 —— 核心解题方法

### 3.1 先搞懂几个基本量

给定一个 IP 地址和前缀 `/N`：

```
总位数    = 32
网络号位  = N（前缀长度）
主机号位  = 32 − N

本网络总地址数    = 2^(32−N)
可用主机地址数    = 2^(32−N) − 2
                 （减 2：第一个是网络地址，最后一个是广播地址）
```

**为什么减 2？**

每个子网（网段）有 2 个特殊地址不能分配给设备：

| 地址 | 主机号部分 | 作用 |
|------|-----------|------|
| 网络地址 | 全 0 | 代表这个网段本身（"这个小区"） |
| 广播地址 | 全 1 | 发给这个网段内所有主机（"全体注意"） |

---

### 3.2 子网划分的本质

子网划分就是：**从原本属于"主机号"的位中，借几位变成"子网号"。**

```
原来： |── 网络号 ──|────── 主机号 ──────|
现在： |── 网络号 ──|── 子网号 ──|── 主机号 ──|
                     ↑ 从主机号借的位
```

**借 m 位后的效果：**

```
子网个数        = 2^m                （m = 借位数）
每子网地址数    = 2^(原主机位数 − m)
每子网可用主机  = 每子网地址数 − 2
新前缀长度      = 原前缀 + m
```

---

### 3.3 步长 —— 子网划分的核心计算

**步长 = 每个子网包含的地址数 = 2^(剩余主机位数)**

步长同时也是**相邻子网起点的差值**。一旦算出了步长，所有子网的范围就一目了然：

```
步长 = 2^(32 − 新前缀) = 2^(剩余主机位数)
```

**步长在掩码中的位置：**

掩码写成四个十进制数，步长 = 256 − 掩码中"非 255 也非 0"的那个数。

```
例：掩码 255.255.255.192
        第四个字节 = 192 → 256 − 192 = 64 → 步长 = 64

例：掩码 255.255.255.224
        第四个字节 = 224 → 256 − 224 = 32 → 步长 = 32
```

---

### 3.4 完整解题流程

**题目：将 `192.168.1.0/24` 划分为 4 个等大于网，写出每个子网的范围和可用 IP。**

```
步骤 1：算借位数
        4 个子网 = 2² → 借 m = 2 位

步骤 2：算新前缀和掩码
        新前缀 = 24 + 2 = 26
        新掩码 = 255.255.255.192
        （192 的来源：前 2 位是 1 → 11000000 = 128+64 = 192）

步骤 3：算步长
        剩余主机位 = 32 − 26 = 6
        步长 = 2⁶ = 64
        或者：256 − 192 = 64

步骤 4：按步长列出所有子网
        子网 1：192.168.1.0    ~ 192.168.1.63   (.0/26)
        子网 2：192.168.1.64   ~ 192.168.1.127  (.64/26)
        子网 3：192.168.1.128  ~ 192.168.1.191  (.128/26)
        子网 4：192.168.1.192  ~ 192.168.1.255  (.192/26)

步骤 5：标记每个子网的网络地址和广播地址
        网络地址 = 范围中第一个（主机号全 0）
        广播地址 = 范围中最后一个（主机号全 1）

        子网 1：网络 192.168.1.0,   广播 192.168.1.63,  可用 .1~.62
        子网 2：网络 192.168.1.64,  广播 192.168.1.127, 可用 .65~.126
        子网 3：网络 192.168.1.128, 广播 192.168.1.191, 可用 .129~.190
        子网 4：网络 192.168.1.192, 广播 192.168.1.255, 可用 .193~.254
```

## 四、再练一题：`172.16.0.0/16` 分成 8 个子网

```
步骤 1：8 个子网 = 2³ → 借 m = 3 位

步骤 2：新前缀 = 16 + 3 = 19
        新掩码 = 255.255.224.0
        （224 的来源：借 3 位给第 3 字节 → 11100000 = 128+64+32 = 224）

步骤 3：剩余主机位 = 32 − 19 = 13
        步长 = 2¹³ = 8192（或者 256 − 224 = 32，步长发生在第 3 字节）

        注意：步长 32 是在第 3 个字节，即 172.16.x.0 中的 x
        每个子网的第 3 字节差 32

步骤 4：按步长列出子网（仅写第 3 字节的变化）
        子网 1：172.16.0.0/19    (172.16.0.0   ~ 172.16.31.255)
        子网 2：172.16.32.0/19   (172.16.32.0  ~ 172.16.63.255)
        子网 3：172.16.64.0/19   (172.16.64.0  ~ 172.16.95.255)
        子网 4：172.16.96.0/19   (172.16.96.0  ~ 172.16.127.255)
        子网 5：172.16.128.0/19  (172.16.128.0 ~ 172.16.159.255)
        子网 6：172.16.160.0/19  (172.16.160.0 ~ 172.16.191.255)
        子网 7：172.16.192.0/19  (172.16.192.0 ~ 172.16.223.255)
        子网 8：172.16.224.0/19  (172.16.224.0 ~ 172.16.255.255)

步骤 5：每子网可用主机 = 8192 − 2 = 8190
```

## 五、关键公式汇总

```
子网个数        = 2^m                          （m = 借位数）
新前缀长度      = 原前缀 + m
步长            = 2^(32 − 新前缀)              （决定子网间隔）
每子网可用主机  = 步长 − 2
掩码最后一个     = 256 − 步长                   （步长在哪个字节步长就减哪个）
```

## 六、三种常见题型

<details>
<summary><b>题型 1：给定网络，分成 N 个子网</b></summary>

> **思路：** 先算借几位（2ᵐ ≥ N）→ 新前缀 → 步长 → 写范围
>
> 例：`192.168.1.0/24`，分 8 个子网
> - 8 = 2³，借 3 位
> - 新前缀 = 27，步长 = 32
> - 子网：`.0/27`, `.32/27`, `.64/27`, ...

</details>

<details>
<summary><b>题型 2：给定网络，要求每子网容纳 N 台主机</b></summary>

> **思路：** 先算主机位需要几位（2ʰ − 2 ≥ N）→ 确定前缀 → 步长
>
> 例：每子网要 50 台主机
> - 2⁶ − 2 = 62 ≥ 50，主机位 = 6
> - 前缀 = 32 − 6 = 26
> - 步长 = 64

</details>

<details>
<summary><b>题型 3：给定 IP 和掩码，判断网段、可用范围</b></summary>

> **思路：** 先确定前缀 → 算步长 → 找到 IP 属于哪一段
>
> 例：`192.168.1.78/26`，步长 = 64
> - 78 ÷ 64 = 1.xxx → 第 2 个子网（编号从 0 开始 = 1）
> - 所在子网：192.168.1.64 ~ 192.168.1.127
> - 可用 IP：192.168.1.65 ~ 192.168.1.126

</details>

## 关键点回顾

- A/B/C 类看第 1 字节：1~126 / 128~191 / 192~223
- 子网划分 = 从主机号借位，借 m 位得 2ᵐ 个子网
- **步长**是核心：决定子网间隔，步长 = 256 − 掩码末字节（或 2^(主机位)）
- 可用 IP = 子网总地址 − 2（网络地址 + 广播地址）

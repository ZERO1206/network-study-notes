# IP 地址分类与子网划分

---

## 传统分类（有类编址）

| 类 | 开头位 | 范围 | 默认掩码 | CIDR | 主机数 |
|----|--------|------|---------|------|--------|
| A | 0xxxxxxx | 1 ~ 126 | 255.0.0.0 | /8 | ~1600万 |
| B | 10xxxxxx | 128.0 ~ 191.255 | 255.255.0.0 | /16 | 65534 |
| C | 110xxxxx | 192.0.0 ~ 223.255.255 | 255.255.255.0 | /24 | 254 |

- 127.x.x.x 保留（127.0.0.1 本机环回）
- D 类 224~239 组播，E 类 240~255 保留

**二进制开头一眼识别：**

```
A: 0.......   →   0.0.0.0 ~ 127.255.255.255
B: 10......   → 128.0.0.0 ~ 191.255.255.255
C: 110.....   → 192.0.0.0 ~ 223.255.255.255
```

---

## 无类编址 CIDR

现实已废弃严格分类，改用 **CIDR**：`IP/前缀长度`，掩码任意长。

```
10.0.0.0/12    → 掩码 255.240.0.0
192.168.1.0/26 → 掩码 255.255.255.192（64 地址小网段）
```

---

## 子网划分实操

**例：`192.168.1.0/24` 分成 4 个等大于网**

```
源网络：/24 → 主机号 8 位 → 256 个地址

分 4 个 = 2² → 借 2 位给网络号
新前缀 = 24 + 2 = 26
新掩码 = 255.255.255.192

每子网 64 地址，可用主机 = 64 - 2 = 62

子网1: 192.168.1.0/26   (.1 ~ .62)
子网2: 192.168.1.64/26  (.65 ~ .126)
子网3: 192.168.1.128/26 (.129 ~ .190)
子网4: 192.168.1.192/26 (.193 ~ .254)
```

**减 2 的原因：** 每个子网第一个地址是网络地址（代表网段本身），最后一个地址是广播地址（发给网内所有主机），不能分配给设备。

---

## 关键公式

```
子网个数   = 2^(借位数)
每子网大小 = 2^(剩余主机位数)
可用主机数 = 每子网大小 - 2
```

---

## 关键点回顾

- A/B/C 类用开头位区分：0 / 10 / 110
- CIDR 替代有类编址，掩码任意长
- 子网划分 = 从主机号借位给网络号
- 可用 IP = 子网大小 − 2（网络地址 + 广播地址）

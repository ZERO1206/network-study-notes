# 第五课：简易聊天室 —— select 多路复用实战

> 把第一课的 echo 服务器升级成多客户端聊天室。核心新知识：`select()` + `fd_set` 四件套。

---

## 一、为什么需要 select

第一课的服务器只能处理一个客户端——`accept()` 阻塞等连接，`recv()` 阻塞等消息，期间第二个客户端完全连不进来。

```
第一课的问题：
accept(server_fd)    ← 堵在这等一个人敲门
recv(client_fd)      ← 堵在这等这个人发消息
                       ↑ 第二个客户端进不来
```

**`select()` 就是解决这个的。** 告诉内核"帮我盯着这几个 fd，哪个有动静了叫我"，一个线程管理多个连接，谁的包先到先处理谁。

## 二、select 和 fd_set 四件套

### fd_set 是什么

`fd_set` 是一个 1024 位的 bitmap。每个 bit 代表一个 fd 编号——bit=1 表示"盯住这个 fd"，bit=0 表示"不管它"。

```
fd_set（简化示意，一行=一个 fd）：
fd=0   fd=1   fd=2   fd=3   fd=4   fd=5   fd=6   fd=7 ...
 [0]    [0]    [0]    [0]    [0]    [0]    [0]    [0]
```

### FD_ZERO——全部清零

```cpp
FD_ZERO(&read_fds);
```

每次 while 循环开始必须调一次。select 返回后会修改 bitmap，下次循环前得重置成干净的。

### FD_SET——把某个 fd 设为"盯住"

```cpp
FD_SET(server_fd, &read_fds);   // 盯 server_fd
FD_SET(client_fd, &read_fds);   // 盯 client_fd
```

把该 fd 对应的 bit 置为 1。

### select()——睡觉等人敲门

```cpp
select(max_fd + 1, &read_fds, nullptr, nullptr, nullptr);
```

| 参数 | 值 | 含义 |
|------|-----|------|
| 第1参 | `max_fd + 1` | 只扫描 fd 0 ~ max_fd，性能优化（不扫整个 bitmap） |
| 第2参 | `&read_fds` | **输入**：盯谁 / **输出**：谁有动静了 |
| 第3参 | `nullptr` | 不监控"可写"事件 |
| 第4参 | `nullptr` | 不监控异常 |
| 第5参 | `nullptr` | 不限时，一直等到有动静 |

**select 会修改 bitmap：只保留"有动静"的 fd 的 1，其他全清零。**

```
select 前：fd=3=1, fd=4=1, fd=5=1  ← "盯着这仨"
    ↓ select()
select 后：fd=3=0, fd=4=1, fd=5=0  ← "只有 fd=4 有消息"

所以每轮循环开头都要重新 FD_ZERO + FD_SET。
```

### FD_ISSET——检查某个 fd 有没有动静

```cpp
if (FD_ISSET(server_fd, &read_fds)) { ... }
if (FD_ISSET(client_fd, &read_fds)) { ... }
```

返回 true = 这个 fd 的 bit 还亮着 = 有数据可读。server_fd 亮 = 有新连接请求，client_fd 亮 = 有消息到达。

### select 循环的生命周期

```
循环开始:
    FD_ZERO()         → bitmap 全 0
    FD_SET(server_fd) → server_fd 置 1
    FD_SET(客户端们)  → 每个客户端 fd 置 1
        ↓
    select()          → 睡觉等，醒来 bitmap 只剩有动静的
        ↓
    FD_ISSET(server)  → 有人敲门？accept
    FD_ISSET(各客户端) → 有消息？recv → 广播
        ↓
    回到循环开头
```

## 三、服务端完整代码

```cpp
#include <iostream>
#include <cstring>
#include <vector>
#include <sys/socket.h>
#include <sys/select.h>    // fd_set / FD_ZERO / FD_SET / FD_ISSET / select
#include <netinet/in.h>
#include <unistd.h>

int main() {
    int server_fd = socket(AF_INET, SOCK_STREAM, 0);

    int opt = 1;
    setsockopt(server_fd, SOL_SOCKET, SO_REUSEADDR, &opt, sizeof(opt));

    sockaddr_in addr{};
    addr.sin_family = AF_INET;
    addr.sin_addr.s_addr = INADDR_ANY;
    addr.sin_port = htons(9999);
    bind(server_fd, (sockaddr*)&addr, sizeof(addr));
    listen(server_fd, 10);    // 比第一课的 3 大，聊天室可能多人同时连

    std::cout << "[聊天室服务器] 监听 0.0.0.0:9999\n";

    std::vector<int> clients;  // 花名册：所有已连接客户端的 fd

    while (true) {
        fd_set read_fds;
        FD_ZERO(&read_fds);
        FD_SET(server_fd, &read_fds);
        int max_fd = server_fd;

        // 把所有客户端 fd 加入监控
        for (int fd : clients) {
            FD_SET(fd, &read_fds);
            if (fd > max_fd) max_fd = fd;
        }

        select(max_fd + 1, &read_fds, nullptr, nullptr, nullptr);

        // 1. 有人敲门——新连接
        if (FD_ISSET(server_fd, &read_fds)) {
            int client_fd = accept(server_fd, nullptr, nullptr);
            clients.push_back(client_fd);
            std::cout << "[聊天室] 新用户加入, fd=" << client_fd
                      << " (共 " << clients.size() << " 人在线)\n";
        }

        // 2. 有客户端发消息
        char buf[1024];
        for (auto it = clients.begin(); it != clients.end(); ) {
            int fd = *it;
            if (FD_ISSET(fd, &read_fds)) {
                int n = recv(fd, buf, sizeof(buf) - 1, 0);
                if (n <= 0) {
                    if (n == 0)
                        std::cout << "[聊天室] 用户离开, fd=" << fd << "\n";
                    close(fd);
                    it = clients.erase(it);   // erase 返回下一个元素
                    continue;                  // 跳过 ++it
                }
                buf[n] = '\0';
                std::cout << "[聊天室] fd=" << fd << " 说: " << buf;

                // 广播给所有人(除了发送者自己)
                for (int other : clients) {
                    if (other != fd) {
                        send(other, buf, n, 0);
                    }
                }
            }
            ++it;   // 只有没被 erase 的才到这里
        }
    }

    close(server_fd);
    return 0;
}
```

编译：`g++ -std=c++17 -o chat_server chat_server.cpp`

### 逐段讲解

#### 为什么用 `std::vector<int>` 存客户端

第一课只有一个客户端，一个 `int client_fd` 就够了。聊天室不知道会有多少人连进来——用 `vector` 动态扩容，`push_back` 新增，`erase` 踢出。

#### `max_fd` 的作用

```cpp
select(max_fd + 1, ...);
```

告诉内核"只扫描 fd 0 ~ max_fd"。纯粹的**性能优化**，和广播逻辑无关。

```
假设当前有 fd=3(server) + fd=4,5,7(客户端):
  max_fd = 7
  → 内核只扫描 fd 0~7，跳过后面 1016 个 bit
```

#### erase + continue 模式

```cpp
for (auto it = clients.begin(); it != clients.end(); ) {
    int fd = *it;
    if (FD_ISSET(fd, &read_fds)) {
        int n = recv(fd, ...);
        if (n <= 0) {
            close(fd);
            it = clients.erase(it);   // erase 返回被删元素的下一个
            continue;                  // 不执行末尾的 ++it
        }
        // ... 广播 ...
    }
    ++it;   // 正常情况手动步进
}
```

**为什么不能 for 头部写 `it++`：** erase 后 `it` 已经指向下一个元素了，再 `++it` 会跳过一个。

| 情况 | it 怎么移 |
|------|----------|
| 正常收到消息 + 广播完 | 执行末尾 `++it`，移向下一个 |
| 客户端断开被踢 | `it = clients.erase(it)` 已经指向下一个，`continue` 跳过 `++it` |

## 四、客户端完整代码

```cpp
#include <iostream>
#include <string>          // std::string / std::getline
#include <cstring>
#include <sys/socket.h>
#include <sys/select.h>
#include <netinet/in.h>
#include <arpa/inet.h>
#include <unistd.h>

int main() {
    int sock = socket(AF_INET, SOCK_STREAM, 0);

    sockaddr_in addr{};
    addr.sin_family = AF_INET;
    addr.sin_port = htons(9999);
    inet_pton(AF_INET, "127.0.0.1", &addr.sin_addr);
    connect(sock, (sockaddr*)&addr, sizeof(addr));

    std::cout << "[聊天室] 已连接! 输入消息回车发送 (输入 /quit 退出)\n";

    while (true) {
        fd_set read_fds;
        FD_ZERO(&read_fds);
        FD_SET(sock, &read_fds);            // 盯服务器消息
        FD_SET(STDIN_FILENO, &read_fds);    // 盯键盘输入
        int max_fd = (sock > STDIN_FILENO) ? sock : STDIN_FILENO;

        select(max_fd + 1, &read_fds, nullptr, nullptr, nullptr);

        // 情况1：服务器发消息来了
        if (FD_ISSET(sock, &read_fds)) {
            char buf[1024];
            int n = recv(sock, buf, sizeof(buf) - 1, 0);
            if (n <= 0) {
                std::cout << "[聊天室] 连接断开\n";
                break;
            }
            buf[n] = '\0';
            std::cout << "[消息] " << buf;
        }

        // 情况2：你敲了键盘
        if (FD_ISSET(STDIN_FILENO, &read_fds)) {
            std::string line;
            std::getline(std::cin, line);
            if (line == "/quit") break;
            line += '\n';
            send(sock, line.c_str(), line.size(), 0);
        }
    }

    close(sock);
    return 0;
}
```

编译：`g++ -std=c++17 -o chat_client chat_client.cpp`

---

### 逐段讲解

#### `STDIN_FILENO`——键盘也是文件描述符

| 宏 | 值 | 是什么 |
|----|-----|--------|
| `STDIN_FILENO` | 0 | 标准输入——键盘 |
| `STDOUT_FILENO` | 1 | 标准输出——屏幕 |
| `STDERR_FILENO` | 2 | 标准错误——屏幕 |

在 Linux 上**键盘和 socket 都是文件描述符**。这就是 select 的精妙之处——一行 `FD_SET(STDIN_FILENO, ...)` 就能让内核同时等待网络和键盘。你不用开两个线程，一个 select 全搞定。

#### 为什么用 `std::string` 而不是 `char[]`

```cpp
std::string line;
std::getline(std::cin, line);
```

键盘输入长度不可预知。`char buf[1024]` 定死了 1023 字节上限，超了就截断甚至溢出。`std::getline` 自动扩容，所以需要 `#include <string>`。

#### `/quit` 退出逻辑

```cpp
if (line == "/quit") break;   // ← 双引号！不是 '/quit'
```

`"/quit"` 是字符串，类型是 `const char[6]`，能和 `std::string` 做 `==` 比较。
`'/quit'` 是**多字符字面量**，类型是 `int`，和 `std::string` 比较时行为未定义，永远不会匹配。

## 六、当前代码的缺陷与改进方向

这个聊天室是学习 select 多路复用的最小可用版本。以下不是 bug，而是**设计上的局限**——了解这些有助于你理解生产级代码要怎么升级。

### 1. select 的 fd 上限：最多 1024 个连接

`fd_set` 底层是固定 1024 位的 bitmap。fd 编号超过 1023 就无法被 `FD_SET` 加入监控。本聊天室理论上最多 1023 个客户端（减去 server_fd）。

**改进方向：** `poll()` 用动态数组替代固定 bitmap，无 fd 上限。`epoll()`（Linux 专用）更进一步，O(1) 事件通知，百万连接也不怕。这是后续要学的内容。

### 2. 广播 send 可能阻塞整个服务端

```cpp
for (int other : clients) {
    if (other != fd) {
        send(other, buf, n, 0);   // ← 如果 other 的接收缓冲区满了，这里会阻塞
    }
}
```

假如某个客户端网速极慢，TCP 发送缓冲区被填满，`send()` 就会阻塞。**一个人的网络卡顿会拖慢所有人的消息。** 当前代码没有任何保护。

**改进方向：** 非阻塞 I/O + 每个客户端独立的发送缓冲区（send queue），send 失败的数据暂存起来下次再试。

### 3. 消息被截断：buf 只有 1024 字节

```cpp
char buf[1024];
int n = recv(fd, buf, sizeof(buf) - 1, 0);  // 最多读 1023 字节
```

客户端发一条超过 1023 字节的长消息，recv 只收前 1023 字节，剩下的数据残留在内核接收缓冲区。下次 select 醒来，`FD_ISSET` 还亮着，但代码接着读——这次读到的不是"下一条消息"，而是**上一条消息的后半截**，导致消息错乱。

**改进方向：** 应用层协议——每条消息前加上长度头（如 4 字节 `uint32_t`），先读长度再读正文，而不是直接把 TCP 流当消息边界。

### 4. TCP 粘包：两次 send 可能被合并成一次 recv

```cpp
// 客户端
send(sock, "hello\n", 6, 0);
send(sock, "world\n", 6, 0);

// 服务端 recv 可能一次收到 "hello\nworld\n" 共 12 字节
```

TCP 是字节流，不是消息流。内核会把多次 `send` 的数据合并到一个 TCP 段里发送，接收方一次 `recv` 可能读到多条消息拼接在一起。

本聊天室每行消息带 `\n` 结尾，恰好用换行符充当了"消息分界"，所以基本不会出问题。但不要误以为 `send` 和 `recv` 是 1:1 对应的。

**改进方向：** 如第 3 点——定义消息边界（长度前缀 / 分隔符 / 定长消息）。

### 5. 没有用户身份——全是 fd 编号

```cpp
std::cout << "[聊天室] fd=" << fd << " 说: " << buf;
```

接收方只看到 `fd=4`、`fd=5`，不知道谁是谁。生产级聊天室在客户端连上后应该发一条"JOIN <昵称>"，服务端记录 `fd → 昵称` 映射，广播时带昵称。

### 6. select 的效率问题：每次循环都要重建 fd_set

```cpp
while (true) {
    FD_ZERO(&read_fds);
    for (int fd : clients) { FD_SET(fd, &read_fds); }  // O(n) 遍历
    select(max_fd+1, ...);  // 内核也要 O(n) 扫描 bitmap
}
```

几百个客户端时感觉不到。几千个以上，每次循环 O(n) 两次（应用层遍历 + 内核扫描）就开始吃 CPU。

**改进方向：** epoll 用事件驱动替代轮询——`epoll_ctl` 一次性注册，`epoll_wait` 只返回有事件的 fd，不需要遍历全部。

### 7. 没有处理 `recv` 返回 < 0 的细分

```cpp
if (n <= 0) {
    // n < 0 直接踢，不区分是临时错误还是真断了
    close(fd);
    it = clients.erase(it);
}
```

`recv` 返回 -1 时，`errno` 可能是 `EAGAIN`（非阻塞 socket 暂时没数据）或 `EINTR`（被信号打断），这两种情况不应该踢人。本聊天室用的是阻塞 socket，所以这个问题不太容易触发，但在生产代码里必须区分。

## 七、自测题

1. `select()` 解决了什么第一课做不到的问题？
2. `FD_SET` 和 `FD_ISSET` 分别干什么？谁在入参、谁在出参？
3. `select` 返回后为什么必须重新 `FD_ZERO` + `FD_SET`？
4. `it = clients.erase(it)` 为什么不用在末尾再 `++it`？
5. `STDIN_FILENO` 是什么？为什么客户端能用 select 同时等键盘和网络？

<details>
<summary>点击查看答案</summary>

1. 第一课的 accept/recv 是阻塞的，一次只能等一个 fd。select 让一个线程同时监控多个 fd，谁的包先到先处理谁。
2. FD_SET 在 select 前调用，把 fd 加入"盯住"名单。FD_ISSET 在 select 后调用，检查某个 fd 是否有动静。FD_SET 是输入参数，FD_ISSET 读的是 select 修改后的输出结果。
3. select 返回时会把没动静的 fd 对应的 bit 全部清零。不重建的话 bitmap 里只剩上一轮有动静的 fd，新连接的 fd 和新消息全漏掉。
4. `clients.erase(it)` 返回被删元素的下一个元素的迭代器。it 已经指向下一个了，再 `++it` 会跳过一个。
5. STDIN_FILENO 是标准输入（键盘）的文件描述符，值为 0。Linux 上 socket 和键盘都是文件描述符，所以 select 能统一监控——服务器来消息 sock 有动静，你敲键盘 stdin 有动静，select 都能醒来。

</details>
